## 0. 현재 상태 스냅샷

**레포에 이미 있는 것**

- Gradle 멀티모듈 8개 (common / domain 5 / app 2) + 의존 방향 선언
- Flyway `V1__init.sql` (스키마 5개, 테이블 21개)
- Docker Compose (로컬 DB + app 2) + Dockerfile 2개
- CI 빌드 워크플로(ci.yml) + VPS 배포 워크플로(deploy.yml, 비활성)
- common: `ApiResponse`(면책 문구 고정) / `TimeUtils`(KST) / `RateLimiter` / `ExternalApiClient`(백오프 재시도) / `DiscordNotifier` + 테스트 7건
- M0 검증 도구: `scripts/m0-kis-verify/kis_verify.py` + `docs/M0-kis-verification.md`

**아직 없는 것 (즉시 액션)**

- [ ] 최초 git 커밋 + GitHub push → CI 통과 확인
- [ ] 의존 방향 1분 체크 (README 참고 — 역참조 컴파일 에러 확인)
- [ ] 빌드 검증 1회 (`./gradlew build` — IntelliJ 또는 CI)

전부 본인 액션 대기. 상세 절차: `docs/M0-kis-verification.md`, 목록: `BACKLOG.md`

- [ ] 한국투자증권 계좌 + KIS 실전 앱키 (순위 TR은 실전 전용 — **가장 오래 걸림, 최우선 신청**)
- [ ] 포털 공지 "신규 고객 초당 호출 제한 (2026.03.20)" 원문 확인
- [ ] `kis_verify.py` 실행: 토큰 → TR 4종 → `--probe` 한도 실측 → J2 예산 판정
- [ ] VPS 계약 → 동일 스크립트로 IP 차단 확인 (차단 시 국내 타 VPS 교체)
- [ ] 나머지 키 발급: OpenDART · 네이버 검색 · FRED · ECOS · KOSIS · Gemini (전부 즉시)
- [ ] LLM 품질 비교(반나절) + 실데이터 1건 프롬프트 검증

**Exit**: TR 4종 성공 + 한도 확정. 한도 부족 시 **여기서** 배치 주기(30→60분)·종목 수(40→25) 조정 후 확정서 반영.

> ⚠️ 발견 사항 (스펙 4단계 수정 필요): 종목별 투자자 TR(FHKST01010900)은 당일치를 장마감 후에만 제공. 장중 잠정 수급은 가집계 TR(FHPTJ04400000, 하루 4~5회 갱신: 09:30/10:00/11:20/13:20/14:30)로 대체 — 30분 배치마다 새 수급이 없을 수 있음을 카드 UI("잠정" 뱃지)와 수급 진단 로직에 반영할 것.

## M2. 원천 데이터 (2주) — domain-market·issue

구현 순서 (의존순):

1. [ ] market: 종목 마스터 동기화(J1) — KIS 종목정보파일
2. [ ] market: daily_price 수집 + **2년치 백필** (일봉+지수 — 도미노 리플레이·베타의 원천)
3. [ ] market: investor_flow — 장중 잠정(가집계 TR) + 마감 확정(inquire-investor) 이원화, vol_ratio_20d 계산
4. [ ] market: stock_beta 주간 계산(J7) — 백필 데이터로 즉시 검증
5. [ ] issue: DART 신규 공시 폴링 + 제목 키워드 룰 분류
6. [ ] issue: 네이버 뉴스 수집 (종목명 쿼리, URL 중복 제거)
7. [ ] collector: J1/J3/J7 스케줄 등록 + 휴장일 캘린더

**Exit**: 평일 하루 무인 방치 → 4개 테이블에 당일 데이터 정상 적재, 잠정→확정 덮어쓰기 확인.
**테스트**: 실응답 JSON 픽스처로 파서 단위 테스트 + "하루 돌려보고 row 수·null 비율 체크" 스크립트.
**공통 기반 활용**: KIS/DART/네이버 클라이언트는 전부 `ExternalApiClient` 상속, 한도는 M0 실측값으로 `RateLimiter` 설정.
