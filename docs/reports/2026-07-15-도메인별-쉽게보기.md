# 도메인별 현황, 쉽게 보기 (초보자용)

> 코드를 잘 몰라도 "**어떤 부분이 무슨 일을 하고, 어떻게 돌아가고, 지금 어디까지 됐는지**"를 알 수 있게 쉬운 말로 정리했어요.
> 정확한 기술 내용·핵심 코드는 아래 두 문서를 보세요.
> - 표로 보는 현황: `2026-07-15-도메인별-구현현황.md`
> - 역할 + 핵심 코드: `2026-07-15-도메인-상세역할-및-핵심코드.md`

## 먼저 큰 그림 — "창고 5개 + 도우미 3개"

이 앱은 데이터를 종류별로 **5개 창고(도메인)**에 나눠 담고, **도우미 3개**가 공통 일을 맡아요.
창고끼리는 서로 함부로 못 건드리게 벽이 쳐져 있어요(그래서 한 창고를 고쳐도 다른 창고가 안 망가짐).
그리고 **참조는 종목코드(6자리)로만** 해요 — 예를 들어 이슈 창고는 시세 창고에 그 종목이 아직 없어도, 종목코드만 알면 공시를 저장할 수 있어요(서로 강하게 묶여있지 않음).

```
        [시세]         [이슈]         [거시]
       market         issue          macro
          │             │              │
          └──────┬──────┴──────────────┘   ← Why 엔진이 읽어옴(고치진 않음)
                 ▼
          🫀 [Why 엔진] insight  ← 이 앱의 심장

        [콘텐츠] content (혼자 독립)      [공통] common (모두가 씀)

   도우미:  common(공통도구) · app-collector(수집) · app-api(화면에 제공)
```

**왜 이렇게 나눴나?** 나중에 특정 창고만 따로 떼어내 별도 서버로 옮길 수 있게 미리 경계를 그어둔 거예요(모듈러 모놀리스). 지금은 1인 운영이라 서버 하나에 다 올리지만, 방향을 어긴 코드는 아예 빌드가 실패하게 막아둬서 경계가 항상 지켜져요.

## 한눈에 보기

| 부분 | 쉬운 이름 | 무슨 일을 하나 | 지금 상태 |
|------|-----------|----------------|-----------|
| `common` | 공통 도구함 | 외부 연결·시간·알림 같은 공통 기능 | 🟢 완료 |
| `domain-market` | 시세 창고 | 주가·거래량·수급·베타 | 🟢 완료 |
| `domain-issue` | 이슈 창고 | 공시·뉴스 | 🟢 완료 |
| `domain-insight` | **Why 엔진(심장)** | 위 창고들을 읽어 "왜?" 카드 생성 | 🟡 진행중 (~70%) |
| `domain-macro` | 거시 창고 | 경제 지표·신호등·날씨 | ⚪ 아직 (예정) |
| `domain-content` | 콘텐츠 창고 | 용어사전·도미노 룰 | ⚪ 아직 (예정) |
| `app-collector` | 수집 도우미 | 30분마다 데이터 모으기 | 🟢 대부분 완료 |
| `app-api` | 제공 도우미 | 화면에 데이터 내어주기 | ⚪ 거의 아직 |

> 🟢 다 만들어짐 · 🟡 만드는 중 · ⚪ 아직 (예정)

## 창고·도우미별 자세한 설명

---

### 🧰 공통 도구함 `common` — 🟢 완료

**하는 일**
모든 창고가 함께 쓰는 기본 도구 모음이에요. 도메인 지식은 없고, "밖과 안전하게 대화하는 법"만 담당해요.

**어떻게 (핵심 도구 3개)**
- **속도 조절기(RateLimiter)**: 증권사(KIS)는 "초당 1번만 불러라"는 제한이 있어요. 이걸 어기면 차단될 수 있어서, 호출 간격을 자동으로 벌려줘요.
- **재시도기(ExternalApiClient)**: 외부 사이트가 잠깐 먹통(429·서버오류·타임아웃)이면 잠깐 기다렸다가 다시 시도해요(기다리는 시간을 점점 늘림). 단, "데이터 없음"과 "진짜 오류"는 구분해서, 진짜 오류만 재시도해요.
- **알림이(DiscordNotifier)**: 배치가 실패하면 디스코드로 알려줘요. 밤에 자동으로 돌다가 문제가 생겨도 아침에 알 수 있게.

**작은 규칙들**: 시간은 항상 한국 시간(KST)으로 통일하고, 모든 화면 응답에는 "이건 투자 조언이 아닙니다" 문구가 자동으로 붙어요.

```
[창고가 외부 API 호출]
      │
      ▼
 속도 조절기 ── 앞 호출과 간격 벌리기(초당 1번)
      │
      ▼
   호출 시도
      ├─ 성공 ───────────────────▶ 결과 반환
      └─ 실패(429·서버오류·타임아웃)
             └─ 잠깐 대기 후 재시도(점점 더 길게)
                    └─ 끝내 실패 ─▶ 예외 + 디스코드 알림
```

**상태**: 다 됐어요. 다른 모든 부분이 이 위에서 돌아가요. 더 손댈 것 없음.

### 📈 시세 창고 `domain-market` — 🟢 완료

**하는 일**
증권사(한국투자증권 KIS)에서 시장 데이터를 가져와 저장하고, Why 엔진이 쓸 계산값을 만들어요. **모든 계산의 재료**가 여기서 나와요.

**어떻게 (모으는 것들)**
- **종목 목록**: 상장된 주식 종목(6자리 코드) 마스터를 동기화
- **매일 주가·지수**: 종목별 일봉(종가·거래량·등락률)과 코스피/코스닥 지수. 오픈할 때 **과거 2년치를 한 번에 채워요(백필)** — 이게 있어야 "과거 그날은 왜 그랬나"를 다시 재현할 수 있어요.
- **수급(누가 사고팔았나)**: 외국인·기관 순매수. 여기엔 함정이 하나 있어요 👉 장중에는 아직 **잠정치(가집계, 하루 4~5번만 갱신)**만 나오고, 진짜 확정치는 **장 마감 후에** 나와요. 그래서 장중엔 '잠정'으로 저장했다가, 마감 후 확정치로 **덮어써요**.

**만드는 계산값 2개** (Why 엔진이 씀)
- **베타(β)**: 시장이 1% 움직일 때 이 종목이 보통 몇 % 움직이는지. 최근 60일 데이터로 계산. → "이 등락 중 시장 몫은 얼마?" 계산에 필요
- **거래량 배율**: 오늘 거래량이 최근 20일 평균의 몇 배인지. → 특징주 "거래량 3배" 판정에 필요

```
[KIS 증권사 API]
   ├─ 종목 목록 ────────────────▶ [종목마스터]
   ├─ 일봉·지수 ────────────────▶ [일별주가] · [지수]
   └─ 수급(외국인·기관)
        장중: 잠정 저장 ──(장 마감 후)──▶ 확정치로 덮어쓰기 ─▶ [수급]

   [일별주가] + [지수] ──▶ 베타·거래량배율 계산 ──▶ 저장
```

**담는 데이터(표)**: 종목마스터, 일별주가, 지수, 수급, 베타 — 5종

**남은 일**: 종목의 **업종 정보**를 아직 못 채웠어요. 종목마스터 원본 파일 안에 업종 코드가 어디 있는지 위치를 확정해야 하는데, 그게 Why 엔진의 "같은 업종이 함께 움직였나" 판단에 필요해서 그 전에 채워야 해요. (그 외에 KIS 연결 코드 중복 정리 정도 — 급하지 않음)

### 📰 이슈 창고 `domain-issue` — 🟢 완료

**하는 일**
급등락의 **이유 후보**를 모아요. Why 카드의 "무슨 일이 있었나"를 뒷받침하는 근거 창고예요.

**어떻게**
- **공시 수집(DART)**: 금융감독원 전자공시(DART)에서 새 공시를 30분마다 확인해요.
  - 상장사 공시만 남기고(비상장 제외), 이미 받은 건 건너뛰어요(같은 걸 두 번 저장 안 함 = 멱등).
  - 제목의 키워드로 종류를 자동 분류해요: **수주계약 / 실적 / 자금조달 / 소송 / 지분변동 / 기타**. 예를 들어 제목에 "공급계약"이 있으면 → '수주계약'.
- **뉴스 수집(네이버)**: 종목 이름으로 기사를 검색해 가져오고, HTML 찌꺼기를 걷어내고, 같은 링크는 중복 제거해요.

**왜 공시를 뉴스보다 중요하게 볼까?** 공시는 회사의 **공식 발표(사실)**이고, 뉴스는 그에 대한 **해석**이라서예요. 그래서 Why 엔진이 근거를 고를 때 공시를 먼저 씁니다.

**안전장치**: 한 건 저장이 실패해도 배치 전체가 죽지 않게 **건별로 격리**해요(한 공시가 이상해도 나머지는 정상 저장).

```
[DART 공시]  ─▶ 상장사·신규만 ─▶ 제목 키워드 분류 ─▶ [공시 저장]
[네이버 뉴스] ─▶ HTML 정제·중복 제거 ─────────────▶ [뉴스 저장]
        (한 건 실패해도 나머지는 계속 = 건별 격리)
```

**담는 데이터(표)**: 공시, 뉴스 — 2종

**남은 일**: 공시가 아주 많은 날 여러 페이지 넘겨 읽기, 뉴스 요약 다듬기 정도. (실무상 급하지 않음)

### 🫀 Why 엔진 `domain-insight` — 🟡 진행중 (~70%) · **이 앱의 심장**

**하는 일**
위 세 창고(시세·이슈·거시)를 **읽어서**(고치진 않음) "이 종목이 왜 움직였나" 카드를 자동으로 만들어요. 이 앱에서 가장 핵심이자 가장 어려운 부분이에요.

**어떻게 (특징주 뽑기 → 4단계 분석)**
먼저 오늘 **눈에 띄게 움직인 종목**을 골라요: 등락률 ±5% 이상 **또는** 거래량 평소 3배 이상 → 그중 거래대금(사고판 금액) 상위 40개. 그다음 각 종목을 4단계로 분석해요.

1. **시장 탓? 이 종목만의 일?** — 오늘 등락에서 "시장 전체가 움직인 몫(= 베타 × 지수 등락)"을 빼봐요. 남는 게 크면 이 종목만의 이유예요.
   - 예: 종목이 −8% 빠졌는데 시장 몫이 −2%면 → 나머지 −6%는 이 종목 개별 사정
2. **같은 업종이 다 같이 움직였나?** — 비슷한 업종 종목들도 함께 빠졌다면, 그 하나의 문제가 아니라 업종 전체 이슈일 수 있어요.
3. **무슨 일이 있었나 (근거)** — 등락 시각 전후의 공시·뉴스에서 이유 후보를 **최대 3건** 골라요. **공시 먼저, 그다음 뉴스**, 각각 최신순.
4. **누가 사고팔았나 (수급)** — 외국인·기관 순매수 방향을 봐요. 금액이 1억 미만이면 '중립'으로 처리(노이즈 제거). 둘이 같은 방향이면 "함께 사들여 우호적", 엇갈리면 "방향이 엇갈림"으로 설명. 장중이면 앞에 '장중 잠정 집계로는,'을 붙여요.

```
[오늘 전 종목]
     │  등락 ±5%↑  또는  거래량 3배↑
     ▼
[특징주 후보] ─▶ 거래대금 상위 40개
     │
     │   (종목마다 4단계)
     ├─ 1. 시장분해 : 등락 − (베타 × 지수) = 개별 몫
     ├─ 2. 업종동조 : 같은 업종도 함께 움직였나
     ├─ 3. 이슈매칭 : 공시 먼저 + 뉴스, 최대 3건
     └─ 4. 수급진단 : 외국인·기관 방향
     ▼
[Why 카드]  (지금=템플릿 문장 / 나중=AI 통역) ─▶ 30분마다 덮어쓰기
```

이 4단계 결과를 모아 카드로 저장하고, **30분마다 자동으로** 다시 만들어요(같은 종목·날짜면 덮어쓰기). 한 종목 분석이 실패해도 나머지는 계속 진행돼요(건별 격리).

**담는 데이터(표)**: Why 카드(feature_card), 체력 진단(health_check — 표만 있고 아직 안 씀)

**지금까지 된 것**: 특징주 뽑기 + 4단계 + 30분 자동 카드 생성까지 전부 동작해요.

**남은 일 (가장 중요) — 3가지**
- 🔴 **AI 통역(3-E)**: 지금은 정해진 **템플릿 문장**으로 카드를 써요("오늘은 지수가 −1.4% 움직인 시장 흐름의 영향이 컸어요…" 같은). 여기에 AI(Gemini)를 붙여 뉴스가 있는 종목은 더 자연스러운 3줄로 만들고, **"사라/팔아라/목표가" 같은 조언성 문구를 걸러내는 필터**와 하루 사용량 상한을 달아야 해요. AI가 실패하면 지금의 템플릿 문장으로 되돌아가요(폴백). → **Gemini API 키만 준비되면 착수**. 붙일 자리는 이미 코드에 마련돼 있어요.
- 🟠 **장기 체력 진단(3-G)**: "이 악재, 오래 갈 이슈인가?" 판단 재료(일회성/구조적 분류·최근 실적 추세·이슈 후 수급 추적) — 아직 표만 있음
- 🟡 **거시 연계(3-H)**: 카드에 시장 배경(신호등·이벤트)을 덧붙이기 — 거시 창고가 생긴 뒤

### 🌍 거시 창고 `domain-macro` — ⚪ 아직 (예정)

**할 일(계획)**
경제 지표 발표를 신호등으로 바꿔주는 창고예요.
- **경제 신호등**: 미국·한국의 지표 14개(고용·물가·금리 등) 발표를 가져와, **발표치 vs 예상치 vs 이전치**를 3단 비교해 🟢좋음 🟡애매 🔴나쁨으로 판정하고 한 줄 해설을 붙여요.
- **시장 날씨**: 코스피 등락 + 공포지수(VIX)를 조합해 '화창~폭풍' 5단계로. 대시보드 배경 테마가 돼요.
- **도미노 실측**: F1 도미노의 "이론상 이렇지만 실제 과거엔 어땠나"를 채워요(이벤트 이력 + 이후 D+1/7/30 자산 반응).
- **예상치**는 크롤링 대신 관리자가 발표 전 수동 입력(약관 리스크 회피).

```
 [발표치]  vs  [예상치(관리자 입력)]  vs  [이전치]
                    │  3단 비교
                    ▼
            🟢 좋음 / 🟡 애매 / 🔴 나쁨  + 한 줄 해설

 [코스피 등락] + [공포지수 VIX] ─▶ 시장 날씨 (화창 ~ 폭풍, 5단계)
```

**상태**: 데이터 담을 자리(표 5개)는 준비돼 있고, **코드는 아직 없어요.** (M4에서 착수)

### 📖 콘텐츠 창고 `domain-content` — ⚪ 아직 (예정)

**할 일(계획)**
초보자용 콘텐츠 창고예요. 다른 창고와 상관없이 독립적으로 동작해요.
- **금융 번역기 용어사전**: 어려운 용어 300개를 쉬운 2문장 설명 + 예문으로. 화면에서 용어에 점선을 긋고 터치하면 말풍선으로 보여줘요. 많이 찾은 용어는 위젯으로.
- **도미노 룰**: "금리 오르면 → 채권은…"처럼 사건이 번지는 경로 규칙 30개. **룰마다 예외 사례 카드 1개 이상 필수**(세상은 이론대로만 안 움직이니까).

**상태**: 자리(표 2개)만 있고 코드는 아직. 대부분 '글을 채우는 작업'이라 화면 개발(M6)과 함께 진행. (M6에서 착수)

### 🕒 수집 도우미 `app-collector` — 🟢 대부분 완료

**하는 일**
정해진 시간에 창고들에게 "데이터 모아와"라고 시키는 알람 시계 역할이에요(스케줄러). 실패하면 디스코드로 알리고, 다음 주기에 다시 시도해서 회복돼요.

**어떻게 (돌아가는 알람들)**

| 언제 | 무엇을 |
|------|--------|
| 평일 오후 4:30 | 종목 목록 + 매일 주가·지수 |
| 장중 30분마다 | 수급(잠정) · Why 카드 생성 |
| 장 마감 후(16:40) | 수급 확정치로 덮어쓰기 |
| 토요일 새벽 6시 | 주간 베타 계산 |
| 30분마다(8~20시) | 공시 · 뉴스 |
| 수동 1회 | 과거 2년치 백필 |

```
 하루 흐름 (평일)
 08~20시 30분마다 : 공시 · 뉴스
 장중 30분마다     : 수급(잠정) · Why 카드
 16:30            : 종목 · 주가 · 지수
 16:40            : 수급 확정치로 덮어쓰기
 토 06:00         : 주간 베타
 (실패하면 디스코드 알림 → 다음 주기에 자동 회복)
```

**남은 일**: 공휴일(주말 말고 '빨간 날') 건너뛰기 달력, 그리고 거시·체력진단이 생기면 그 알람들도 추가.

### 🖥 제공 도우미 `app-api` — ⚪ 거의 아직 (예정)

**하는 일(계획)**
프론트 화면과 관리자에게 데이터를 내어주는 창구(REST API)예요.

**할 일**
- **조회 API 8종**: 특징주 카드 목록, 체력 진단, 신호등, 용어사전, 도미노 룰/리플레이 등. 자주 보는 목록은 캐시(임시 저장)로 빠르게.
- **관리자 API 5종**: 예상치 입력, 검수 등 (관리자 토큰으로 보호).

```
 [프론트 화면] ──요청──▶ [app-api] ──조회──▶ [창고들: market·issue·macro·insight·content]
                            └ 자주 보는 목록은 캐시로 빠르게 응답
 [관리자] ──관리자 토큰──▶ [app-api /admin]
```

**상태**: 지금은 "서버 살아있나요?" 확인용 하나만 있어요. 다만 **데이터베이스 표 설계(21개)는 이 부분이 관리** 중이라 이미 준비 완료. 조회·관리자 API는 M4에서. (프론트 화면 M5를 시작하려면 이게 먼저 있어야 해요)
