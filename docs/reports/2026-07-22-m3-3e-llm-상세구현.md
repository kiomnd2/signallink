## 1. 무엇을, 왜

시그널링크의 핵심은 "이 종목이 **왜** 올랐/내렸는지"를 초보자 언어로 설명하는 것이다. 그 문장이
`feature_card.what_happened`다. 기존 J2 파이프라인(3-F)은 이미 카드를 생성하지만, `what_happened`가
**시장분해 템플릿 문장 전용**이었다.

```java
// (기존) FeatureCardPipelineService — LLM 자리를 비워두고 항상 템플릿·false
market.whatHappened(),           // 템플릿 문장
... false, CardStatus.PUBLISHED  // llm_used = false 고정
```

3-E는 이 자리에 **LLM(Gemini)로 생성한 자연스러운 설명**을 끼우되, 실패하면 조용히 템플릿으로
되돌아가도록(폴백) 만드는 작업이다. 즉 "LLM이 있으면 더 좋은 문장, 없거나 실패해도 최소한 템플릿"이라는
**graceful degradation** 구조.

## 2. 전체 아키텍처 (헥사고날 레이어 배치)

```
app-collector (인바운드: FeatureCardJob 스케줄러)
        │  generate(tradeDate)
        ▼
┌─────────────────────────── domain-insight ───────────────────────────┐
│                                                                       │
│  application/service                                                  │
│    FeatureCardPipelineService  ── 종목 루프 + 예외 격리 (조립 지휘)   │
│         │  WhyContext 조립                                            │
│         ▼                                                             │
│    WhySummaryService  ── 게이트(활성/이슈/상한) · 필터 · 폴백 판정     │
│         │  compose(prompt)              ▲ Optional<String>            │
│         ▼                               │                             │
│  domain (순수)                          │                             │
│    WhyPromptComposer ─ 프롬프트 조립    │                             │
│    BannedWordFilter  ─ 조언성 차단 ─────┘                             │
│         │  generate(prompt)                                           │
│         ▼                                                             │
│  application/port/out                                                 │
│    LlmGatewayPort  (인터페이스)  ◄─── 의존 역전 경계                  │
│         ▲ 구현                                                        │
│  adapter/out/external/gemini                                          │
│    GeminiClient (ExternalApiClient 상속) ─ HTTP ─▶ Gemini API         │
│    GeminiRequest / GeminiResponse (레코드)                            │
└───────────────────────────────────────────────────────────────────────┘
```

**레이어 규칙 준수**
- `domain`(BannedWordFilter·WhyContext·WhyPromptComposer)은 프레임워크·기술 의존 **0**. 단독 단위 테스트.
- `application`은 포트(`LlmGatewayPort`)에만 의존 — Gemini를 모른다.
- `adapter/out/external`만 기술(RestClient·Jackson·Gemini 스키마)에 의존.
- 의존 역전: 서비스 → `LlmGatewayPort`(포트) ← `GeminiClient`(어댑터). LLM 교체(OpenAI 등) 시 어댑터만 추가.

## 3. 데이터 흐름 (한 종목 처리 한 사이클)

```
FeatureCandidate (선정된 특징주 1건)
   │
   ├─ MarketDecomposeService.analyze() → MarketAnalysis(marketContrib, driver, sectorSync, whatHappened=템플릿)
   ├─ FlowDiagnoseService.diagnose()   → FlowDiagnosis(flowSummary, provisional)
   └─ IssueMatchService.match()        → IssueMatch(refs: 공시/뉴스 최대 3건)
        │
        ▼  파이프라인이 위 3개를 WhyContext로 조립
   WhyContext(stockCode, changeRate, triggerType, marketContrib, driver, sectorSync, flowSummary, issues)
        │
        ▼  WhySummaryService.summarize(ctx, tradeDate)
   ┌── enabled=false?        ──▶ Optional.empty()  ┐
   ├── issues 비었나?        ──▶ Optional.empty()  ├─▶ (폴백) whatHappened = 템플릿, llm_used=false
   ├── 일 상한 초과?         ──▶ Optional.empty()  │
   │                                               │
   ├─ WhyPromptComposer.compose(ctx) → prompt      │
   ├─ LlmGatewayPort.generate(prompt) ─────────────┤ 예외 → catch → Optional.empty() (폴백)
   │      (GeminiClient: HTTP POST → JSON 파싱)     │
   ├─ 응답 blank?            ──▶ Optional.empty()  │
   ├─ BannedWordFilter 금지어? ──▶ Optional.empty() ┘
   └─ 클린 요약 ──▶ Optional.of(summary)  ──▶ whatHappened = summary, llm_used=true
        │
        ▼
   FeatureCardUpsert(... whatHappened, ... llm_used ...) → cardRepository.upsert()
```

핵심: `summarize()`는 **어떤 이유로든 실패하면 항상 `empty()`**를 돌려주고, 파이프라인은
`llmSummary.orElseGet(market::whatHappened)`로 템플릿 폴백한다. 카드 생성이 LLM 때문에 멈추지 않는다.

## 4. 컴포넌트별 상세

### 4.1 BannedWordFilter (순수 도메인) — 조언성 이중 차단의 2막

기획상 서비스는 "왜 움직였나"만 말하고 **투자 조언(매수/매도/목표가)은 절대 안 넣는다.** 프롬프트로 1차
차단하지만, LLM이 규칙을 어길 수 있으니 **출력을 한 번 더 검사**한다.

**함정 — 오탐**: 수급 설명에는 "외국인이 **순매수**했어요", "기관 **매수세**"처럼 `매수`/`매도`가 정상적으로
들어간다. 단순히 `contains("매수")`로 막으면 **정상 카드가 매번 폴백**된다. 그래서 스캔 전에 사실 표현을 제거한다.

```java
// 조언이 아닌 사실 표현(수급 등) — 스캔 전에 제거해 오탐을 막는다.
private static final List<String> FACTUAL_ALLOW = List.of(
    "순매수", "순매도", "매수세", "매도세", "매수량", "매도량",
    "매수 규모", "매도 규모", "매수액", "매도액", "매수 우위", "매도 우위");

private static final List<String> ADVISORY = List.of(
    "매수", "매도", "사세요", "파세요", "사라", "팔아라", "팔아",
    "목표가", "목표주가", "적정주가", "적정가", "투자의견", "추천", ...);

public static boolean hasAdvisory(String text) {
    if (text == null || text.isBlank()) return false;
    String scan = text;
    for (String factual : FACTUAL_ALLOW) scan = scan.replace(factual, "");  // 사실 먼저 제거
    for (String w : ADVISORY) if (scan.contains(w)) return true;           // 남은 것에 금지어?
    return false;
}
```

- `"외국인이 순매수했어요"` → `순매수` 제거 → 남은 문장에 `매수` 없음 → **clean**
- `"지금 매수하기 좋아요"` → 제거 대상 없음 → `매수` 검출 → **advisory**

### 4.2 WhyContext + WhyPromptComposer (순수 도메인) — 프롬프트 설계

`WhyContext`는 3-B~3-D 결과를 모은 **사실 묶음**. 프롬프트는 이 사실만 넣어 **모델이 지어내지 않도록** 한다.

```java
public record WhyContext(
    String stockCode, BigDecimal changeRate, TriggerType triggerType,
    BigDecimal marketContrib, Driver driver, boolean sectorSync,
    String flowSummary, List<IssueRef> issues) {}
```

`WhyPromptComposer`는 (1) 역할·규칙(조언 금지·초보자 언어·2~3문장), (2) 사실 목록을 조립한다. 결정적
(deterministic) 함수라 프롬프트에 사실/규칙이 들어갔는지 단위 테스트로 검증 가능.

```
[규칙]
- 매수/매도/목표가 등 투자 조언이나 전망은 절대 쓰지 않습니다.
- 사실에 없는 내용을 지어내지 않습니다.  ...
[사실]
- 오늘 등락률: +6.20%
- 시장 vs 개별 요인: 시장 몫(+2.00%p)을 넘어서는 개별 요인이 큼
- 수급(외국인·기관): 외국인은 순매수, 기관은 순매도예요.
- 관련 이슈:
  - [공시] 단일판매·공급계약체결
```

`marketContrib`가 null(베타·지수 부족)이면 "개별 움직임으로 봄", 이슈가 없으면 "매칭된 공시·뉴스 없음"으로
분기 — 데이터 공백을 프롬프트가 정직하게 표기한다.

### 4.3 LlmGatewayPort (out 포트) — 의존 역전 경계

```java
public interface LlmGatewayPort {
    /** 오류·차단·빈 응답이면 RuntimeException. 호출측이 잡아 템플릿 폴백. */
    String generate(String prompt);
}
```

계약이 단순한 이유: **"조용한 실패" 금지** 원칙(DART 어댑터와 동일). 빈 응답을 `""`로 돌려주면 상위가
"LLM이 빈 설명을 줬다"고 착각한다. 그래서 문제가 있으면 **예외**로 알리고, 서비스가 명시적으로 폴백한다.

### 4.4 WhySummaryService — 게이트·필터·폴백의 오케스트레이터

3단 게이트 → 호출 → 필터, 그리고 모든 실패 경로가 `empty()`로 수렴한다.

```java
public Optional<String> summarize(WhyContext ctx, LocalDate tradeDate) {
    if (!enabled) return Optional.empty();                       // ① 키 발급 전 비활성
    if (ctx.issues() == null || ctx.issues().isEmpty())
        return Optional.empty();                                 // ② 근거 이슈 없는 종목은 생략(비용·품질)
    if (!tryConsumeBudget(tradeDate)) return Optional.empty();   // ③ 거래일 일 상한

    try {
        String summary = llm.generate(WhyPromptComposer.compose(ctx));
        if (summary == null || summary.isBlank()) return Optional.empty();
        summary = summary.strip();
        if (BannedWordFilter.hasAdvisory(summary)) return Optional.empty();  // 출력 필터
        return Optional.of(summary);
    } catch (RuntimeException e) {
        log.warn("LLM 요약 실패, 템플릿 폴백: {} ({})", ctx.stockCode(), e.toString());
        return Optional.empty();                                 // 전송/파싱 오류도 폴백
    }
}

// 거래일이 바뀌면 카운터 리셋 → 하루 최대 daily-limit회만 LLM 호출
private synchronized boolean tryConsumeBudget(LocalDate tradeDate) {
    if (!tradeDate.equals(budgetDate)) { budgetDate = tradeDate; used = 0; }
    if (used >= dailyLimit) return false;
    used++; return true;
}
```

- **일 상한**은 거래일(`tradeDate`) 기준 인메모리 카운터. 30분 배치가 하루 여러 번 돌아도 총 호출을
  `daily-limit`(기본 60)로 제한 → 비용 폭주 방지.
- 생성자는 `@Value`로 `enabled`·`daily-limit`을 받는 **수동 생성자**(CLAUDE.md: @Value 특수 생성자는 수동
  유지). 덕분에 테스트에서 `new WhySummaryService(stub, true, 1)`로 직접 구성 가능 → 상한/폴백 단위 테스트.

### 4.5 GeminiClient (어댑터) — ExternalApiClient 위의 HTTP 배선

공통 기반 `ExternalApiClient`(rate limit + 지수 백오프 재시도)를 상속하고 `LlmGatewayPort`를 구현한다.
DART·네이버 클라이언트와 동일한 패턴.

```java
@Component
public class GeminiClient extends ExternalApiClient implements LlmGatewayPort {
    public GeminiClient(
        @Value("${signallink.llm.gemini.base-url:https://generativelanguage.googleapis.com}") String baseUrl,
        @Value("${signallink.llm.gemini.api-key:}") String apiKey,
        @Value("${signallink.llm.gemini.model:gemini-3-flash}") String model,
        @Value("${signallink.llm.gemini.permits-per-second:1}") double pps) {
        super(baseUrl, new RateLimiter(pps), 3, 500);
        ...
    }

    @Override public String generate(String prompt) {
        GeminiResponse res = execute("gemini.generate", () -> client().post()
            .uri(uri -> uri.path("/v1beta/models/" + model + ":generateContent")
                .queryParam("key", apiKey).build())
            .contentType(MediaType.APPLICATION_JSON)
            .body(GeminiRequest.of(prompt))
            .retrieve().body(GeminiResponse.class));
        String text = res == null ? null : res.firstText();
        if (text == null || text.isBlank())
            throw new IllegalStateException("Gemini 생성 텍스트 없음(차단/빈 응답): ...");
        return text;
    }
}
```

**세부 결정**
- **경로의 콜론** `models/{model}:generateContent` — Spring UriBuilder가 비-첫 세그먼트의 `:`를
  인코딩하지 않아 그대로 전송된다(HTTP 테스트로 `encodedPath == "/v1beta/models/test-model:generateContent"`
  확인). `%3A`로 깨지면 실 API가 404를 낸다.
- **응답은 평문 요청**: 모델에 평문 2~3문장을 받고 **응답 JSON 엔벨로프만** `GeminiResponse`로 파싱한다.
  내부에서 또 JSON을 요구하지 않아 어댑터가 우리 프롬프트의 세부 스키마에 묶이지 않는다("JSON 파싱"=엔벨로프).
- **`@JsonIgnoreProperties(ignoreUnknown=true)`**: 수동 `RestClient`는 Spring Boot가 설정한 ObjectMapper(
  unknown 무시)를 자동으로 받지 않을 수 있어, 응답 레코드에 명시 → `safetyRatings`·`usageMetadata` 등 무시.
- **키 미설정이어도 빈 생성 가능**: `@Value` 기본값(`api-key:` 빈 문자열)으로 app-api 등 스캔만 하는 곳에서도
  기동 실패 없음. 실제 호출은 `enabled=true`일 때만 서비스가 수행.

```java
public record GeminiRequest(List<Content> contents, GenerationConfig generationConfig) {
    public static GeminiRequest of(String prompt) {
        return new GeminiRequest(List.of(new Content(List.of(new Part(prompt)))),
            new GenerationConfig(0.4, 512));   // temperature, maxOutputTokens
    }
    ...
}
```
레코드 컴포넌트명이 그대로 camelCase JSON 필드(`maxOutputTokens` 등)가 되어 Gemini 스펙과 일치.

### 4.6 파이프라인 배선

```java
WhyContext ctx = new WhyContext(c.stockCode(), c.changeRate(), c.triggerType(),
    market.marketContrib(), market.driver(), market.sectorSync(), flow.flowSummary(), issues.refs());
Optional<String> llmSummary = whySummaryService.summarize(ctx, c.tradeDate());
String whatHappened = llmSummary.orElseGet(market::whatHappened);   // 성공 시 LLM, 아니면 템플릿

cardRepository.upsert(new FeatureCardUpsert(..., whatHappened, ..., llmSummary.isPresent(), PUBLISHED));
```

`whySummaryService`는 종목 루프 안에서 호출되므로 **기존 종목 단위 예외 격리**(try/catch)가 그대로 LLM
경로도 보호한다 — 게다가 서비스 자체가 내부에서 폴백하므로 예외가 위로 새지도 않는다.

## 5. 설정 (app-collector/application.yml)

```yaml
signallink:
  llm:
    enabled: ${LLM_ENABLED:false}            # 기본 off — 키 발급 전 템플릿 유지
    daily-limit: ${LLM_DAILY_LIMIT:60}
    gemini:
      base-url: https://generativelanguage.googleapis.com
      api-key: ${GEMINI_API_KEY:}
      model: ${GEMINI_MODEL:gemini-3-flash}  # 실호출 검증 시 정확한 ID 확인 후 덮어쓰기
      permits-per-second: ${GEMINI_PPS:1}
```

**기본 off 이유**: 키가 없는 상태에서 켜두면 매 종목마다 인증 실패 → 백오프 재시도 → 결국 폴백이라 낭비다.
꺼두면 게이트 ①에서 즉시 폴백. 키를 발급받으면 `LLM_ENABLED=true` + `GEMINI_API_KEY=...` 환경변수만
주면 활성화 — **코드 변경/재빌드 없이 운영 스위치로 전환**.

## 6. 안전장치 요약

| 위험 | 방어 |
|------|------|
| LLM이 투자 조언 문구 생성 | 프롬프트 금지 명시 + 출력 `BannedWordFilter` (이중 차단) |
| 수급 사실("순매수")을 조언으로 오탐 | 필터가 사실 표현을 스캔 전 제거 |
| LLM 비용 폭주 | (1) 이슈 없는 종목 생략 (2) 거래일 일 상한(기본 60) |
| LLM 장애/타임아웃/차단 | 예외 → 서비스 catch → 템플릿 폴백 (카드 생성 계속) |
| 키 미설정/미검증 모델 | 기본 `enabled=false` → 기존 동작 유지, 운영 스위치로만 on |
| 외부 API rate limit | `ExternalApiClient` 상속(RateLimiter + 지수 백오프) |

## 7. 테스트 전략 & 커버리지 (20건, 스킵 0)

- **순수 도메인은 실호출 없이 완전 검증** — 필터 오탐/정탐, 프롬프트 사실·규칙 삽입.
- **서비스는 스텁 포트로 모든 분기** — 비활성·이슈없음·클린·금지어·예외·빈응답·상한초과·거래일 리셋.
- **어댑터는 MockWebServer로 HTTP 경로** — 실키·실호출 없이 경로/모델/키/바디 조립 + 응답 파싱 + 후보없음 예외.

- **파이프라인은 LLM 성공(llm_used=true)/폴백(false) 양쪽** + 기존 예외 격리 회귀.

## 8. 실행 시나리오

| 상황 | 결과 |
|------|------|
| 키 없음(기본) | 게이트 ① → 폴백. 템플릿 문장, `llm_used=false` |
| 활성 + 이슈 없는 종목 | 게이트 ② → 폴백(LLM 호출 안 함) |

| Gemini가 "매수 추천" 반환 | 필터 검출 → 폴백(조언성 문구 카드 미노출) |
